# 盤面の数値化（特徴量エンコーディング）

AIへの入力は `app/feature_encoder.py` で2種類のテンソルに変換されます。
すべての特徴量は **現在の手番プレイヤー視点** でエンコードされます（先手・後手で座標を反転）。

---

## FEATURES1  `(1, 62, 9, 9)` — 盤上の駒情報

62チャンネル = 自駒31ch + 相手駒31ch の対称構造。

| チャンネル | 内容 |
|---|---|
| 0–13 | 自駒の位置（駒種別 × 9×9 盤面） |
| 14–27 | 自駒の利き（その駒が攻撃できるマス） |
| 28–30 | 自駒の利き数カウント（1枚以上 / 2枚以上 / 3枚以上） |
| 31–44 | 相手駒の位置 |
| 45–58 | 相手駒の利き |
| 59–61 | 相手駒の利き数カウント |

駒種とチャンネルオフセットの対応:

| offset | 駒 | offset | 駒 |
|---|---|---|---|
| 0 | 歩 | 7 | 王 |
| 1 | 香 | 8 | と |
| 2 | 桂 | 9 | 杏 |
| 3 | 銀 | 10 | 圭 |
| 4 | 角 | 11 | 全 |
| 5 | 飛 | 12 | 馬 |
| 6 | 金 | 13 | 龍 |

**座標変換**（`board[row][col]` → テンソルの `[fi][ri]`）:

```
先手の場合:  fi = 8 - col,  ri = row
後手の場合:  fi = col,      ri = 8 - row   （盤面を180度回転）
```

---

## FEATURES2  `(1, 57, 9, 9)` — 持ち駒・王手フラグ

57チャンネル = 自持ち駒28ch + 相手持ち駒28ch + 王手フラグ1ch。

持ち駒の枚数は **温度計エンコーディング**: 歩を3枚持っていれば ch0, ch1, ch2 が全マス 1 になります。

| チャンネル | 駒 | 最大枚数 |
|---|---|---|
| 0–7 | 歩 | 8 |
| 8–11 | 香 | 4 |
| 12–15 | 桂 | 4 |
| 16–19 | 銀 | 4 |
| 20–23 | 金 | 4 |
| 24–25 | 角 | 2 |
| 26–27 | 飛 | 2 |
| 28–55 | 相手の持ち駒（同様） | — |
| 56 | 王手フラグ（全マス 0 or 1） | — |

---

## エンコーディング全体の流れ

```
盤面状態
  │
  ├─ board_2d[9][9]        ──→  FEATURES1 (1, 62, 9, 9)
  │   駒の位置と利き               手番視点で座標変換
  │
  ├─ hands {"black": {...}} ──→  FEATURES2 (1, 57, 9, 9)
  │   持ち駒の枚数                  温度計エンコーディング
  │
  └─ in_check (bool)       ──→  FEATURES2 ch56

      ↓ concat (119ch)

  WideResNet10
      ↓
  ├─ Policy (2187,)  = 27方向 × 81マス → 指し手確率
  └─ Value  (1,)     = tanh スカラー → 先手有利(+1) ～ 後手有利(-1)
```
